# Project instructions

When working on onboarding, activation, signup, first-run, or retention flows:

- Always use the `onboarding-architect` skill first.
- Infer platform from the repository before proposing UI.
- Read README, docs, app copy, routes, and key feature modules before writing screen copy.
- Treat onboarding as context-aware product work, not generic UI work.
- Before generating code, present:
  - platform profile
  - inferred product intent
  - aha moment
  - checklist of onboarding modules
- Reuse existing navigation, styling, motion, and state management.
- Prefer native-feeling UX for iOS, Android, React Native, and Flutter.
